# detail 命令展示本地 skill 文件规格

> 文档类型：SPEC
> 状态：已完成
> 更新时间：2026-03-02
> 适用范围：skuare-cli, docs
> 关联计划文件路径：plan/2026-03-02-09/detail-command.md

## 背景与目标
- 当前 `skr` 已支持远程查询、安装、发布和本地写入类命令，但缺少一个直接查看本地 skill 文件内容的轻量命令。
- 用户要求新增 `skr detail [skillRelativePath...]`，用于展示当前 skill 目录下指定相对路径文件的内容；当未传参数时，默认展示 `SKILL.md`。
- 目标是在不引入远程依赖和复杂交互的前提下，为本地 skill 调试与阅读提供直接入口，并同步更新帮助和文档。

## 需求范围
- 新增 `detail` 命令，接收 0 到多个 `skillRelativePath` 位置参数。
- 当参数为空时，命令默认读取并展示当前目录下的 `SKILL.md` 内容。
- 当参数非空时，按给定相对路径读取当前目录下对应文件内容并输出。
- 若一次读取多个文件，输出需要能够区分每个文件对应的内容。
- 更新 CLI help、README 与必要测试。

## 非目标
- 不引入远程仓库查询、模糊搜索或版本解析能力。
- 不在本需求中新增文件编辑、分页、语法高亮或交互式查看能力。
- 不改变现有其他命令的输出结构和参数语义。

## 用户验收标准与风险
- 验收标准：
  - 在 skill 目录内执行 `skr detail` 时，会输出 `SKILL.md` 内容。
  - 执行 `skr detail refs/a.md docs/b.txt` 时，会按路径展示对应文件内容，且能明确区分不同文件。
  - 对不存在文件、绝对路径或越界相对路径会给出清晰错误。
  - CLI 帮助、README、测试与构建结果已同步。
- 风险：
  - 若直接允许任意路径读取，可能读取到 skill 目录外文件，造成越界访问。
  - 多文件连续输出若没有清晰分隔，用户难以判断内容归属。

## 需求优化建议
- 建议将 `detail` 限制为仅接受当前 skill 目录内的相对路径，并拒绝绝对路径及 `..` 越界路径；收益是行为可预测且更安全；代价是不能顺便查看目录外文件；风险是少量用户若依赖“顺手读外部文件”的隐式行为，需要改为在目标目录执行命令。
- 建议多文件输出时增加统一文件头分隔；收益是结果可读性更高；代价是输出不再是纯原始拼接；风险是若用户已有脚本直接消费纯内容，需要适配新的分隔格式。
