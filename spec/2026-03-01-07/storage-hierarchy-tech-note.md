# 分级存储机制技术说明规格

> 文档类型：SPEC
> 状态：已完成
> 更新时间：2026-03-01
> 适用范围：docs, skuare-cli, skuare-svc
> 关联计划文件路径：plan/2026-03-01-07/storage-hierarchy-tech-note.md

## 背景与目标
- 当前仓库已有 `docs/tech_summary.md` 作为技术综述，但“分级存储机制”分散在 CLI 配置、本地局部仓库、服务端远程仓库等多个实现位置，缺少一份可直接引用的专题说明。
- 用户要求将现有分级存储机制写入 `docs/` 目录，作为正式技术说明。
- 目标是沉淀一份与当前实现一致的技术文档，明确层级边界、目录结构、数据流与已废弃的错误认知。

## 需求范围
- 在 `docs/` 下新增专门说明分级存储机制的技术文档。
- 文档需覆盖：
  - 服务端远程仓库存储根与目录结构；
  - CLI 配置分级与合并优先级；
  - CLI 本地局部仓库的 `global/workspace/tool/skill` 层级；
  - `publish` 与 `get` 的主要数据流；
  - 当前边界：服务端存储目录只由服务端启动参数决定。
- 在现有技术综述中增加对此专题文档的引用入口。

## 非目标
- 不修改现有存储实现与目录结构。
- 不在本需求中新增示意图生成脚本、架构图图片或对外网站文档。
- 不扩展服务端 API 或 CLI 配置字段。

## 用户验收标准与风险
- 验收标准：
  - `docs/` 下存在一份独立的“分级存储机制”说明文档。
  - 文档内容与当前代码实现一致，不再出现 `remote.storageDir` 由 CLI 控制的错误表述。
  - `docs/tech_summary.md` 能引导读者跳转到该专题文档。
  - CLI 构建检查与服务端全量测试通过。
- 风险：
  - 若文档表述过度抽象，可能与具体目录结构脱节。
  - 若引用未更新，读者仍可能只看到综述而错过专题说明。

## 需求优化建议
- 建议后续为 `docs/` 补充统一索引页，把 `tech_summary`、`skill_deps_format`、`storage hierarchy` 等专题文档统一挂载。
- 收益：降低文档发现成本。
- 代价：需要维护文档导航。
- 风险：若目录继续扩张，索引页也需要持续更新。
