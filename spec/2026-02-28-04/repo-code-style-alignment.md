# 仓库代码风格对齐规格

> 文档类型：SPEC
> 状态：进行中
> 更新时间：2026-02-28
> 适用范围：project-wide
> 关联计划文件路径：plan/2026-02-28-04/repo-code-style-alignment.md

## 背景与目标
- 当前仓库在 CLI/SVC 命令命名、默认路径、脚本入口和文档示例上存在多处历史语义残留，导致同一能力在不同入口表现不一致。
- 需要做一轮仓库级代码风格与语义对齐，优先修复影响开发者理解和日常使用的明显不一致项。
- 目标是在不引入功能回归的前提下，统一主入口命名、默认路径文案和脚本/文档风格，使仓库行为更一致、可维护。

## 需求范围
- 对齐 `Makefile`、CLI 帮助、README、脚本等入口中的命令命名，统一使用当前有效语义。
- 清理明显不合理的代码风格问题，如重复声明、过期常量默认值、命名与行为不一致的帮助文本。
- 保持现有构建与测试命令可用，并在修改后完成全量验证。

## 非目标
- 不做大规模重构。
- 不改变后端 API 协议。
- 不在本需求中引入新的格式化工具或全面重排整个代码仓库。

## 用户验收标准与风险
- 用户验收标准：
  - 仓库主入口（`Makefile`、CLI help、README）不再混用 `create/publish`、旧路径和过期说明。
  - 明显的重复声明和过时默认值被清理。
  - CLI 与 SVC 全量测试/构建通过。
- 风险：
  - 入口文案对齐可能影响依赖旧命令说明的用户习惯。
  - 对 `Makefile` 的语义修正若处理不当，可能影响本地开发脚本行为。

## 需求优化建议
- 建议后续引入统一 lint/format 流程（如 ESLint/Prettier + gofmt 校验入口），把“风格一致性”变成自动约束而不是人工清理。
