# skuare 演进路线图

> [English Version](./roadmap.md)

> 文档类型：ROADMAP
> 状态：已完成
> 更新时间：2026-03-10
> 适用范围：project-wide

## 当前定位

`skuare` 当前已经具备本地优先 Skill Registry 的基础闭环：

- `skuare-cli` 提供本地 Skill 编辑、依赖文件生成、远端查询、远端发布、更新、删除、安装与按需依赖安装。
- `skuare-svc` 提供基于文件系统的远端仓库、版本化读写、校验、重建索引与签名鉴权。
- 项目已经形成 “本地编辑 -> 远端发布 -> 本地安装 -> 依赖管理” 的完整链路。

现阶段更准确的产品定位，不是“若干命令的集合”，而是“Skill 包管理与分发基础设施”的 MVP。

## 当前短板

从现有实现看，后续最值得补强的不是继续增加零散命令，而是把以下平台底座能力做实：

- 版本治理仍有一部分依赖 CLI 补偿逻辑，尚未完全下沉到服务端统一约束。
- `author` 已出现在资源展示中，但还没有成为稳定的命名空间和权限边界。
- 依赖模型已可用，但服务端还不能原生理解、校验和索引完整依赖图。
- 检索仍以简单查询、正则和客户端选择器为主，扩展到更大规模后会受限。
- 存储层当前偏单机文件系统模式，更适合本地/轻量协作，而非平台化部署。

## 路线原则

- 先做平台底座，再做体验增强。
- 先统一服务端规则，再扩展 CLI 行为。
- 一个里程碑对应一类能力闭环，便于继续拆分独立需求。
- 中长期方向保持弹性，不在路线图阶段过早锁死实现方案。

## 里程碑

### M1：服务端版本治理收敛

目标：把版本比较、`maxVersion`、版本递增约束与更新前置条件下沉到服务端统一治理。

范围：
- 服务端统一版本排序与比较规则
- `publish/update` 的版本前置条件统一
- CLI 从服务端消费一致的版本判断结果

验收重点：
- 不同客户端对同一 Skill 的版本判断结果一致
- 推荐版本与最大版本不再依赖客户端私有逻辑

### M2：作者与权限模型升级

目标：把当前松散的 `metadata.author` 与签名鉴权提升为真正的作者命名空间和写权限模型。

范围：
- 明确 `author/name` 的资源语义
- 建立 `key_id` 与 author 的授权关系
- 更新、删除、覆盖等写操作按 author 做权限判定

验收重点：
- 非授权 key 无法操作其他 author 名下资源
- author/name 在 CLI 与服务端中保持一致

### M3：依赖治理与索引能力

目标：把现有依赖文件和 `wrap/deps` 工作流推进为服务端可理解、可校验、可查询的依赖治理体系。

范围：
- 服务端依赖闭包检查
- 依赖索引与依赖树查询
- 发布前依赖存在性与一致性校验
- 安装与按需拉取规则统一

验收重点：
- 依赖缺失、版本不一致、循环依赖等场景有稳定反馈
- `publish/get/deps` 的依赖语义一致

### M4：检索与发现能力增强

目标：把当前基于 `q`、正则和客户端交互选择的检索体验，升级为可规模化使用的服务端发现能力。

范围：
- name/author/tag/description 等多维检索
- 服务端过滤与排序能力
- latest/stable/canary 等版本视图或选择能力

验收重点：
- Skill 数量增加后仍可稳定定位目标
- 客户端不再承担过多补充型筛选逻辑

### M5：存储后端与平台化能力

目标：在保留文件系统模式的前提下，为多环境部署、恢复与审计建立后端扩展路径。

范围：
- 保留 FS store 作为本地/开发模式
- 规划更适合平台场景的存储后端
- 增强审计、回收、修复与索引重建能力

验收重点：
- 不同存储实现下 CLI 行为保持一致
- 仓库恢复与修复链路可操作

### M6：产品体验与运维辅助

目标：在底座稳定后，补足诊断、状态检查、发布辅助和可视化入口等效率能力。

范围：
- 配置/环境/连通性诊断
- 本地与远端状态对比
- 发布提示、升级提示与排障入口
- 视情况建设轻量 Web 浏览或只读控制面

验收重点：
- 用户无需深入代码即可定位常见配置与运行问题
- 体验增强不破坏前面里程碑形成的治理规则

## 推荐起步顺序

若只选择一个近期立项方向，优先从 `M1` 开始。

原因：
- 当前 `update` 已经暴露出版本规则在客户端补偿实现的问题。
- 版本规则若不先统一，后续权限、依赖和检索都容易建立在不稳定前提上。
- `M1` 完成后，可以直接为 `M2`、`M3`、`M4` 提供统一的资源版本基础。
